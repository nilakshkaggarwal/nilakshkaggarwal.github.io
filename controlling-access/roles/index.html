    User Roles

    There are 2 types of access or authorization that needs to be defined for the user

    1. UI Access: A person needs UI access to see the workspace or UI elements
    2. Data Access: Data access allow user to see data in the dimensions or measures

    It's the combination of both the roles that allow user to see the Reports in the Workspace.

    Defining Roles

    1. Go to the User Management (Users) workspace,
    2. Go to 'Manage User' tab click on little arrow button before the Tenant name.
    3. Click on AUTHORIZATION GROUPS
    4. Open the Tab UI/Data for which you need to define the role
    5. Add new record to make a UI role and click on update.

    Note: Different Role name should be used for UI and DATA roles

    Assigning Roles

    Now these roles need to be assigned to the user

    1. In the Manage User only
    2. Go to the USER tab
    3. Assign the role in the authorization group column of the particular user.
    4. Click on the save changes button.

    Workspace Authorization

    Workspace needs to be authorized to access by providing them UI role. Implication the particular UI role has access to this Workspace

    1. Click on the layout button and the open the workspace tab 2.Click on the 3 dots at the top right of the workspace icon. 3.Next click on the edit option, under the roles section include the newly made role (Role1) that has been given to the user.

    For DATA role one needs to control access by writing Data Security Rules
